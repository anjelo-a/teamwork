# Purpose

Keep the interface readable, usable, and uncluttered.

# When to use

Use when:

* editing layouts
* editing spacing, sizing, alignment, or responsiveness
* changing typography or visual hierarchy
* working on cards, forms, tables, modals, dashboards, or navigation
* improving interaction clarity

# When not to use

Do not use when:

* making backend-only changes
* editing docs only
* making non-visual infra or config changes

# Required behavior

* prioritize readability over visual clutter
* keep spacing sufficient
* maintain clear hierarchy
* ensure mobile usability
* avoid cramped layouts and overflow
* preserve accessibility and touch-friendliness
* keep content density reasonable

# Forbidden behavior

* cramped layouts
* unreadable typography
* poor mobile fit
* hidden overflow issues
* overly dense UI with weak hierarchy

# Delivery checklist

* layout readable on common screen sizes
* spacing and typography are clear
* no obvious overflow or cramping
* interaction targets remain usable
